How this page works

A data platform light enough to fit inside a web page.

There's no server doing this. No database, no API behind a login, nothing to page me at 3am.

Every morning a GitHub Action runs a small dlt → dbt → DuckDB pipeline — extract, model, test — and commits one Parquet file plus the JSON this page draws. The site is static, served from Cloudflare's edge; the pipeline, its models and its run logs are all public in the repo. And when you press Run up there, that Parquet loads into DuckDB compiled to WebAssembly and every chart query executes right here, in your browser. My heavier, private services run on my own Hetzner box behind WireGuard. Right tool, right weight, for each job.

Don't take my word for it — the data is a public endpoint. Query this site from your own terminal:
duckdb -c "SELECT tool, value AS stars
FROM read_parquet('https://rensevanderzee.nl/data/metrics.parquet')
WHERE mart = 'stars_current' ORDER BY stars DESC;"
About

I'm currently building the data platform at Winparts, a fast-growing European car-parts retailer.

Before that I contributed to one of the largest data migrations in the Netherlands — the mortgage migration at a.s.r. / Aegon — through Sogeti, first as a data engineer and before that as a test analyst.

MSc in Technology & Operations Management and BSc in Information Science, both from the University of Groningen.

I have a bias: right-sized over heavyweight. By day that means BigQuery with dlt and dbt in containers — at Winparts' scale, a managed warehouse is the boring choice. This site runs the other end of the spectrum: a fully open-source stack light enough to live in CI. The point is never the tool; it's matching the weight of the platform to the size of the problem.

How I work
01

Right-sized, not rebuilt

Most workloads don't need a cluster. DuckDB, Parquet and a single box go a long way — and when you do outgrow them, a managed warehouse like BigQuery is the boring next step. Right-sizing is the skill.

02

Pipelines in CI, not on call

Scheduled jobs belong in version control and CI, where they're reviewable and reproducible — not on a server someone has to babysit at 3am.

03

Boring that stays up

Fewer moving parts, clearer failure modes, less to page you about. The best infrastructure is the kind you stop thinking about.
