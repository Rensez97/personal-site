How this page works

    
A data platform light enough to fit inside a web page.

    

      
There's no server doing this. No database, no API behind a login, nothing to page me at 3am.

      
Every morning a GitHub Action runs a small dlt → dbt → DuckDB pipeline and commits one Parquet file. The site is static — served from Cloudflare's edge. When you pressed Run, that Parquet loaded into DuckDB compiled to WebAssembly, and every query executed right here, in your browser. My heavier services — scrapers, a little CRM — run on my own Hetzner box behind WireGuard. Right tool, right weight, for each job.

How I work

    

      

        
01

        
Right-sized, not rebuilt

        
Most workloads don't need a cluster. DuckDB, Parquet and a single box go a long way before you reach for anything heavier — and cost a fraction to run.

      

      

        
02

        
Pipelines in CI, not on call

        
Scheduled jobs belong in version control and CI, where they're reviewable and reproducible — not on a server someone has to babysit at 3am.

      

      

        
03

        
Boring that stays up

        
Fewer moving parts, clearer failure modes, less to page you about. The best infrastructure is the kind you stop thinking about.
